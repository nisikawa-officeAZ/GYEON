# 06 — LINE REQUIREMENTS

> Canonical inputs: `gyeon_settings_flow.json` (line, line-api, auto-message, reminder-templates) and `gyeon_flow.json` (STEP5 「LINE転送」). Implementation: `src/app/api/line/*`, `src/lib/line/*`.

---

## 1. Canonical intent (from JSON)

### 1.1 Estimate forwarding (from `gyeon_flow.json`)
STEP5 action **「LINE転送」 → `shareToLine()`** — send the estimate as text via LINE.

### 1.2 Greeting/notice text (from settings `line`, group g5, 準備中)
- `line-header`, `line-footer` — header/footer for estimate-forward messages.
- `maint-header`, `maint-footer` — header/footer for maintenance notices.

### 1.3 Official-account integration (settings `line-api`, 準備中)
- Channel ID, Channel Secret, Webhook URL (auto-generated by app).
- **Saved to server (D1) only — never KV.**

### 1.4 Auto-message (settings `auto-message`, 準備中) — server-side only.

### 1.5 Reminder (settings `reminder-templates`)
3 templates (1/6/12-month), each with a LINE `message` body; used to send maintenance reminders. See `03` §5.8.

> Canonical status of all LINE settings is **準備中 (hidden)** — i.e. specced but intended as a later activation.

---

## 4. Security rule (carry-over from implementation policy)
`LINE_CHANNEL_SECRET` and `LINE_CHANNEL_ACCESS_TOKEN` are **server-side only** and must never be exposed in the UI. Only safe public metadata (LIFF ID, Webhook URL, QR, connection status) may be displayed. This matches the canonical "line-api saved to server only" rule.

## 5. Operator decisions
1. Activate group **g5** (currently 準備中/hidden) — decide when LINE settings become visible/editable in-app.
2. Provision LINE credentials (channel + token + LIFF) to make §3 operational.
3. Confirm reminder scheduling mechanism (cron/edge function + `CRON_SECRET`).
